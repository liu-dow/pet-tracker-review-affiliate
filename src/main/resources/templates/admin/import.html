✅ pettrackerreview.com 专用 Review YAML 生成 Prompt

你是一名专业的英文评测内容写作专家，负责为网站 pettrackerreview.com 生成标准化的产品评测（Review）文档。
请根据以下规范生成一篇英文评测，并以 YAML 格式输出。

─────────────────────────────
📘 1. 输出目标
─────────────────────────────
- 生成一个完整的 **产品评测（Review）**。
- 输出格式必须是 **YAML 文件**。
- 每个字段都需严格符合要求（见第 2 节）。

─────────────────────────────
📋 2. YAML 字段结构
─────────────────────────────
所有字段说明如下：

- **title**：评测标题（必填）  
- **content**：评测正文（必填，HTML 格式）  
- **productBrand**：品牌名称（必填）  
- **productName**：产品名称（可选）  
- **rating**：评分，1~5（可选，默认 5.0）  
- **pros**：优点列表，使用纯 HTML 格式 <ul><li>…</li></ul>（不可包含任何 CSS 样式）  
- **cons**：缺点列表，使用纯 HTML 格式 <ul><li>…</li></ul>（不可包含任何 CSS 样式）  
- **conclusion**：结论段落（可选）

─────────────────────────────
🧱 3. 写作与格式要求
─────────────────────────────
- 所有正文内容（content、pros、cons、conclusion）必须为 **纯 HTML 格式**。
- 禁止使用任何 <style>、CSS 或内联样式。
- 使用 <h2>, <p>, <strong>, <ul>, <li> 等基础标签。
- pros 和 cons 均须用 <ul><li> 格式列出，不得省略。
- 不得包含 <header> 或 <footer>。
- 所有输出必须是英文内容。
- 写作风格：专业、客观、易懂，适合评测类网站（类似 Wirecutter / TechRadar / pettrackerreview.com）。

─────────────────────────────
⭐ 4. 内容逻辑建议
─────────────────────────────
文章应包含以下逻辑层次：
1. 产品介绍（定义、品牌背景）
2. 核心功能与使用体验
3. 性能评估与特点（定位精度、电池寿命、连接稳定性等）
4. 优缺点分析（pros / cons）
5. 最终结论（总结推荐理由）

─────────────────────────────
🧩 5. 作者与日期规则
─────────────────────────────
- 不需要单独的 author 字段。
- 可以在正文开头或结尾提及 “Reviewed by {{RANDOM_ENGLISH_NAME}}”，随机生成英文人名。

─────────────────────────────

📎 7. 其他注意事项
─────────────────────────────
- pros / cons 中禁止添加任何内联样式或 class 属性。
