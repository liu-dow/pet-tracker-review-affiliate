 AI Content Generation Guidelines

                
                

                    
Blog Post Structure

                    
When generating blog content, ensure it follows this YAML structure:

                    

                        
                        
---
title: "Your Blog Title Here"
author: "Author Name"
date: "YYYY-MM-DD HH:MM:SS"
tags:
  - "tag1"
  - "tag2"
  - "tag3"
content: |
  <h2>Main Heading</h2>
  <p>Your content here...</p>
  
  <h3>Subheading</h3>
  <p>More content...</p>
  
  <ul>
    <li>Bullet point 1</li>
    <li>Bullet point 2</li>
  </ul>
slug: "url-friendly-slug"
metaDescription: "SEO meta description (150-160 characters)"


                    

                

                
                

                    
Review Structure

                    
When generating review content, ensure it follows this YAML structure:

                    

                        
                        
---
title: "Product Name Review: Key Feature Highlighted"
author: "Reviewer Name"
date: "YYYY-MM-DD HH:MM:SS"
tags:
  - "brand"
  - "product type"
  - "feature"
metaDescription: "SEO meta description (150-160 characters)"
slug: "product-name-review"
productName: "Product Name"
productBrand: "Brand Name"
rating: 4.5
pros: |
  <ul>
    <li>Pro 1</li>
    <li>Pro 2</li>
    <li>Pro 3</li>
  </ul>
cons: |
  <ul>
    <li>Con 1</li>
    <li>Con 2</li>
    <li>Con 3</li>
  </ul>
conclusion: |
  Brief conclusion summary (2-3 sentences)
content: |
  <h2>Review Title</h2>
  <p>Introduction paragraph...</p>
  
  <h3>Feature Analysis</h3>
  <p>Detailed feature analysis...</p>
  
  <h3>Final Verdict</h3>
  <p>Final recommendation...</p>


                    

                

                
                

                    
Writing Guidelines

                    
    
                        
  • SEO Optimization: Include relevant keywords naturally throughout the content
    • 
                        
  • Readability: Use clear headings, bullet points, and short paragraphs
    • 
                        
  • Objectivity: Provide balanced pros and cons for reviews
    • 
                        
  • Technical Accuracy: Ensure all technical specifications are accurate
    • 
                        
  • Engagement: Write in a conversational tone that connects with pet owners
    • 
                        
  • Length: Blog posts should be 800-1200 words, reviews 600-1000 words
    • 
                        
  • Special Note: Review pros and cons must use <ul><li> format as shown in the example above
    • 
                    

                

                
                

                    
Recommended AI Prompts

                    
Use these prompts as templates for generating content:

                    
                    
Blog Prompts:

                    
    
                        
  • "Write a comprehensive blog post about [TOPIC] for pet owners. Include sections on key features, comparison of top products, and buying guide. Use a friendly, informative tone."
    • 
                        
  • "Create an SEO-optimized blog post about [PET_TRACKER_TYPE] for [PET_TYPE]. Focus on [KEY_FEATURES] and include practical advice for pet owners."
    • 
                    

                    
                    
Review Prompts:

                    
    
                        
  • "Write a detailed, unbiased review of the [PRODUCT_NAME] by [BRAND]. Include pros, cons, performance analysis, and a final recommendation. Use a conversational tone for pet owners."
    • 
                        
  • "Create a comprehensive review of [PRODUCT_NAME]. Analyze [ASPECTS_TO_COVER] and provide a balanced assessment with a rating out of 5 stars."
    • 
                    

                

                
                

                    
Important Format Requirements

                    
Please pay special attention to the following format requirements:

                    
    
                        
  • Pros/Cons Format: The pros and cons fields in reviews MUST use <ul><li> HTML format
    • 
                        
  • Date Format: Use "YYYY-MM-DD HH:MM:SS" format
    • 
                        
  • Tag Format: Format tags as a YAML array with each tag on a separate line starting with a dash
    • 
                        
  • Content Format: Use appropriate HTML tags including headings, paragraphs, and lists
